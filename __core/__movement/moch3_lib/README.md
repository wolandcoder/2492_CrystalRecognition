# Mach3 Remote Control System

Система для удалённого управления станком ЧПУ под управлением Mach3 с помощью Python. Позволяет отправлять команды перемещения, запрашивать координаты и автоматизировать работу станка через локальную сеть.

## Архитектура

┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Клиент        │     │   Сервер        │     │   Mach3         │
│ (mach3_client)  │────▶│ (mach3_server)  │────▶│  Macropump      │
│                 │◀────│                 │◀────│                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
     Генерация               Запись/чтение            Управление
     G-кода                  файлов                   станком

- **Клиент** формирует объект `MyData`, генерирует строку G-кода и отправляет её на сервер.
- **Сервер** принимает строку, записывает в файл `Mach3Command.txt`, ожидает завершения движения и возвращает текущие координаты.
- **Macropump** в Mach3 постоянно читает этот файл, выполняет команды и обновляет файл `DROData.txt` с координатами.

## Требования

- **Mach3** версии R3.043.066 или новее
- **Python 3.7+** на клиенте и сервере
- Библиотеки Python (устанавливаются автоматически):
  - pickle (встроенная)
  - socket (встроенная)
  - struct (встроенная)

## Установка

1. Склонируйте репозиторий или скопируйте файлы проекта на клиентский компьютер и на компьютер с Mach3.

2. На обоих компьютерах убедитесь, что Python установлен и добавлен в PATH.

3. Дополнительные зависимости не требуются — используются только стандартные модули.

## Настройка Mach3 и Macropump

Для взаимодействия с Mach3 используется механизм **Macropump** — фоновый макрос, выполняющийся циклически.

1. Создайте папку для файлов обмена (например, `B:\test\data\` или `C:\Mach3\Data\`). Убедитесь, что у Mach3 есть права на запись в неё.

2. В папке макросов вашего профиля Mach3 (обычно `C:\Mach3\macros\[ИмяПрофиля]\`) создайте файл `Macropump.m1s` со следующим содержимым:

```basic
' Macropump.m1s – Двусторонний обмен: запись координат и выполнение команд
Dim fso, file, strLine, commandFile, command
Dim XPos, YPos, ZPos, APos

XPos = GetDRO(0)
YPos = GetDRO(1)
ZPos = GetDRO(2)
APos = GetDRO(3)

Set fso = CreateObject("Scripting.FileSystemObject")

' Запись координат в DROData.txt
Set file = fso.CreateTextFile("B:\test\data\DROData.txt", True)
strLine = XPos & "," & YPos & "," & ZPos & "," & APos
file.WriteLine(strLine)
file.Close

' Обработка входящих команд
commandFile = "B:\test\data\Mach3Command.txt"
If fso.FileExists(commandFile) Then
   Set file = fso.OpenTextFile(commandFile, 1)
   If Not file.AtEndOfStream Then
      command = file.ReadLine
      file.Close
      fso.DeleteFile(commandFile)
      If Len(command) > 0 Then
         Code(command)   ' Выполнить G-код
      End If
   Else
      file.Close
   End If
   Set file = Nothing
End If
Set fso = Nothing
```

3. Перезапустите Mach3. Macropump активируется автоматически.

4. Убедитесь, что файл `DROData.txt` создаётся и обновляется при изменении координат.

## Запуск сервера

На компьютере, к которому подключён станок с Mach3, запустите серверный скрипт:

```bash
python mach3_server.py
```

По умолчанию сервер слушает порт `5555` на всех интерфейсах (`0.0.0.0`). Пути к файлам можно изменить внутри скрипта.

## Библиотека `mach3_common.py`

Это основной модуль, содержащий класс `MyData` и вспомогательные функции для работы с ним.

### Класс MyData

Представляет данные для отправки на станок. Состоит из двух вложенных классов:

- **`Motion`** – параметры движения
- **`Connection`** – сетевые настройки

#### Создание объекта

```python
from mach3_common import MyData

# С указанием IP и порта
data = MyData(ip='192.168.1.124', port=5555)

# Без параметров (можно задать позже)
data = MyData()
data.connection.set_ip('192.168.1.124').set_port(5555)
```

#### Установка координат и подачи

```python
data.motion.set_x(10.5)      # X = 10.5 мм
data.motion.set_y(20.0)      # Y = 20.0 мм
data.motion.set_z(-1.0)      # Z = -1.0 мм
data.motion.set_feed(700)    # скорость подачи 700 мм/мин
```

Методы можно вызывать цепочкой:

```python
data.motion.set_x(10).set_y(20).set_feed(500)
```
Так же есть возможность двигаться относительно текущей точки

```python
data.motion.move_on(axis, data)
```
где axis это ось "x""y""z""a" с маленькой буквы, а дата это float число на сколько переместиться

#### Установка готовой G-код команды

```python
data.motion.set_command("G0 X10 Y20 Z5")
```

Если задана `command`, она имеет приоритет при генерации G-кода.

Так же есть возможность использовать готовые наборы команд через 
```python
data.motion.send_command(command_name)
```

где в поле command_name мы передаем одну из этих строк
"tozero" - убрать накопленную ошибку (перейти в координаты -1 -1 -1)
"fullcollibration" - полностью отколибровать станок (некоторое время будет шуметь до коллибровки)
"allzero" - сделать текущую точку центром координат

#### Чтение значений

```python
x = data.motion.X
y = data.motion.Y
cmd = data.motion.command
```

#### Сброс параметров движения

```python
data.motion.clear()
```

### Генерация G-кода

Функция `generate_gcode(data: MyData) -> str` преобразует объект `MyData` в строку G-кода.

**Приоритет:**
1. Если задано поле `command`, возвращается оно без изменений.
2. Иначе собирается строка из координат `X`, `Y`, `Z`, `A` и подачи `f`.
   - Если задана хотя бы одна координата, используется `G0` (без подачи) или `G1` (если указан `f`).

Примеры:

```python
from mach3_common import MyData, generate_gcode

data = MyData()
data.motion.set_x(10).set_y(20).set_feed(500)
print(generate_gcode(data))   # "G1 X10.000 Y20.000 F500"

data.motion.clear().set_command("M3 S1000")
print(generate_gcode(data))   # "M3 S1000"
```

### Отправка команд на сервер

Функция `send_command_string(data_obj: MyData, expect_response=True) -> str` генерирует G-код и отправляет его на сервер.

```python
from mach3_common import MyData, send_command_string

data = MyData(ip='192.168.1.124', port=5555)
data.motion.set_x(50).set_y(0)

response = send_command_string(data)
print(response)  # Например, "50.000,0.000,-5.000,0.000"
```

Возвращаемое значение — строка с координатами после выполнения команды, либо сообщение об ошибке.

## Интерактивный клиент

Скрипт `mach3_client.py` предоставляет консольный интерфейс для ручного управления.

Запуск:

```bash
python mach3_client.py
```

Основные команды:

| Команда | Описание |
|---------|----------|
| `set ip <адрес>` | Установить IP сервера |
| `set port <порт>` | Установить порт сервера |
| `set x <число>` | Задать координату X |
| `set y <число>` | Задать координату Y |
| `set z <число>` | Задать координату Z |
| `set a <число>` | Задать координату A |
| `set f <число>` | Задать скорость подачи (мм/мин) |
| `set cmd <текст>` | Задать готовую G-код команду |
| `show` | Показать текущие значения |
| `clear motion` | Сбросить все параметры движения |
| `gcode` | Показать сгенерированную строку G-кода |
| `send` | Отправить команду на станок |
| `get` | Запросить текущие координаты |
| `set_command` | выполнить комманду из списка |
| `quit` | Выход |

## Примеры использования

### 1. Простое перемещение по осям

from mach3_common import MyData, send_command_string

data = MyData(ip='192.168.1.100', port=5555)
data.motion.set_x(100).set_y(50).set_feed(800)

response = send_command_string(data)
print(f"Приехали в точку: {response}")

### 2. Запрос координат

data.motion.clear().set_command("GET_DRO")
coords = send_command_string(data)
print(coords)

### 3. Обнуление координат

data.motion.clear().set_command("G92 X0 Y0 Z0")
send_command_string(data)


## Устранение неполадок

### Сервер не запускается / ошибка подключения

- Убедитесь, что на сервере разрешён входящий трафик на порт `5555` в брандмауэре.
- Проверьте, что IP-адрес сервера доступен с клиента (команда `ping`).
- Убедитесь, что серверный скрипт запущен и не выдаёт ошибок.

### Команды не выполняются, файл `Mach3Command.txt` не появляется

- Проверьте права на запись в папку, указанную в `COMMAND_FILE`.
- Убедитесь, что Macropump активен (перезапустите Mach3).
- Проверьте пути в `Macropump.m1s` и в серверном скрипте — они должны совпадать.

### Координаты не обновляются / возвращается `NO_DATA`

- Убедитесь, что файл `DROData.txt` создаётся Macropump'ом и содержит актуальные данные.
- Проверьте путь к файлу в серверном скрипте.

### Ошибка `pickle` или декодирования

- Убедитесь, что на сервере и клиенте используется совместимый протокол (отправляется строка, а не объект класса).
- Обновите `mach3_common.py` и сервер до последней версии из этого репозитория.
